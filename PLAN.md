# 《听见北京》项目规划

> Beijing / Between Axis and Breath
> 一个以声音地图为核心的极简北京城市介绍网页

---

## 一、项目概述

| 项目 | 说明 |
|------|------|
| **名称** | 听见北京 / Beijing Between Axis and Breath |
| **选题** | 方向 B：城市介绍 |
| **形式** | 单页网页 (One-page website) |
| **技术** | 原生 HTML + CSS + JavaScript（无框架） |
| **部署** | GitHub Pages / 临时发布链接 |
| **截止** | 7月19日 23:59 |

### 核心理念
不做普通旅游宣传网页，而是做一个**有数字展览感的城市声音交互网页**。用户通过滚动、点击和声音，感受北京的古都秩序、胡同烟火、现代都市和夜晚气息。

### 差异化记忆点
- **声音地图**作为核心互动，而非附加功能
- **进度收集机制**（0/4 → 4/4）创造探索完成感
- **极简东方美学**，区别于 AI 泛化审美

## 三、页面结构（7 个 Section）

### 01 Hero 首屏
- **目的**：一打开就有高级感，建立"展览"基调
- **内容**：
  - 项目名：听见北京
  - 英文副标题：Beijing / Between Axis and Breath
  - 一句说明文案
  - "开启声音体验"按钮（解决浏览器自动播放限制）
- **视觉**：
  - 米白背景 + 大量留白
  - 中间偏左排版
  - 细红线作为视觉引导
  - 背景极淡的中轴线抽象线条（SVG，opacity < 0.05）
- **文案**：
  > 一条中轴线，延展出古都的秩序；
  > 无数街巷与灯火，构成今天北京的呼吸。

### 02 北京的坐标 / 概况
- **目的**：用数据建立认知，不堆百科
- **内容**：4 个数据卡片
  1. 三千多年建城史
  2. 八百余年建都史
  3. 中国首都
  4. 文化中心 / 科技创新中心
- **视觉**：
  - 左侧标题 `01 / 北京的坐标`
  - 右侧四个小卡片（CSS Grid）
  - 卡片只放数字、关键词和一句说明
- **文案**：
  > 北京是一座被时间层层叠加的城市。它既有古都的轴线、城门与礼制，也有地铁、写字楼和不断生成的新生活。

### 03 看见北京 / 地标与风景
- **目的**：视觉冲击，4 张地标卡片
- **内容**：
  1. 故宫 — 红墙之内，时间像一层安静的回响。
  2. 天坛 — 北京与天空之间，保留着古老的仪式感。
  3. 胡同 — 真正的北京，常常藏在热闹背后的转角里。
  4. CBD — 玻璃、速度与灯光，构成城市的未来表面。
- **视觉**：
  - 四张图片卡片（2×2 Grid，移动端单列）
  - 默认低饱和（filter: saturate(0.7)）
  - hover 轻微放大 + 饱和度恢复
  - 点击可播放对应声音（与声音地图联动）
  - 每张卡片只保留标题和一句话

### 04 理解北京 / 人文与特色
- **目的**：三个关键词，不写空泛文化底蕴
- **内容**：
  - **秩序** — 中轴线不只是城市结构，也是北京精神的一种可视化表达。
  - **包容** — 不同口音、身份和梦想在这里相遇，塑造了今天的北京。
  - **反差** — 皇家建筑旁边可以是早餐铺，写字楼不远处也能转进老胡同。
- **视觉**：
  - 三列横排（移动端竖排）
  - 每段前有大号编号（01 02 03）
  - 红色只用于编号
  - 留白多，文字短

### 05 靠近北京 / 美食与生活
- **目的**：时间线形式展示"北京的一天"
- **内容**：
  - 07:00 清晨 — 豆汁、焦圈、胡同口的早餐铺，城市在热气里醒来。
  - 12:30 午间 — 一碗炸酱面，简单、直接，也很北京。
  - 18:30 傍晚 — 铜锅涮肉升起白雾，冬天的北京有了温度。
  - 22:00 夜晚 — 后海的水声、街灯和晚风，让城市慢下来。
- **视觉**：
  - 纵向时间线（左侧时间，右侧内容）
  - 时间数字用宫墙红
  - 每个时间点配小图
  - 滚动进入时轻微动画

### 06 声音地图 / 核心互动 ⭐
- **目的**：项目最大亮点，收集四种北京气质
- **内容**：4 个声音卡片
  1. **中轴北京** — 故宫、中轴线、古都秩序 | 钟声/古琴/风声
  2. **胡同北京** — 胡同、街巷、清晨生活 | 自行车铃/鸟鸣/人声
  3. **都市北京** — CBD、地铁、科技创新 | 地铁提示/电子节拍/白噪声
  4. **夜色北京** — 后海、街灯、晚风 | 水声/晚风/低频 ambient
- **反馈机制**：
  - 初始状态：`探索进度 0/4`
  - 每点击一个卡片：进度 +1，显示反馈文案
  - 卡片高亮（边缘细线）
  - 点击其他卡片：停止当前音频，播放新音频
  - 四个全部点亮后显示总结：
    > 你已听见一个更完整的北京。
    > 它不只有宏大，也有细小而真实的温度。
- **视觉**：
  - 2×2 Grid 卡片布局
  - 进度条在顶部（宫墙红填充）
  - 卡片有播放图标状态
  - 已收集的卡片有标记

### 07 未来寄语
- **目的**：收尾，留下余韵
- **内容**：
  > 如果你还没来过北京，愿这座城市先以声音和光影与你相遇。
  > 如果你已经来过，也希望你记得它不只有宏大，还有细小而真实的温度。
- **视觉**：
  - 全屏大留白
  - 居中排版
  - 底部项目署名 + GitHub Pages 链接

## 四、交互设计详细

### 4.1 声音系统
```
用户点击卡片
  → 停止当前播放的音频
  → 播放该卡片对应音频
  → 卡片高亮（边缘细线动画）
  → 如果首次点击：进度 +1
  → 显示反馈文案（底部 toast 或卡片内）
  → 4/4 完成：显示总结文案
```

### 4.2 滚动动效
- Intersection Observer 监听 section 进入视口
- 进入时：文字 fade-in + translateY(20px → 0)
- 缓动函数：cubic-bezier(0.25, 1, 0.5, 1) (ease-out-quart)
- 持续时间：600ms

### 4.3 响应式断点
```css
/* 移动优先 */
/* 默认：单列布局 */

/* 平板 ≥ 768px */
@media (min-width: 768px) {
  /* 部分双列 */
}

/* 桌面 ≥ 1024px */
@media (min-width: 1024px) {
  /* 完整多列布局 */
}
```

## 八、Impeccable 设计检查清单

- [ ] **AI Slop Test**：展示给别人说"这是 AI 做的"，他们会相信吗？
- [ ] 字体：Noto Serif SC + Inter，非 Inter/Roboto 单一字族
- [ ] 色彩：无纯黑 #000 / 纯白 #fff，无紫蓝渐变，无霓虹
- [ ] 空间：有节奏变化的间距，非万物皆卡片
- [ ] 动效：状态变化动效，指数缓动，无回弹
- [ ] 响应式：容器查询 / 媒体查询，移动端不隐藏功能
- [ ] 文案：每个词有价值，不重复可见信息
- [ ] 细节：有意图的装饰，无毛玻璃滥用
